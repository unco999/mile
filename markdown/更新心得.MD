目前 GPU DSL 这条链路上真正“打通”的是条件表达式；仍然有几块功能/支线还没实现或只有雏形，可以留意：

build_connections (mile_gpu_dsl/src/mat/gpu_ast.rs (line 1236)) 仍旧没被调用，意味着旧的矩阵规划→ GPU AST 精确接线逻辑还未整合；现在依赖的是 convert_to_final_output_ast 内部的较直接构造。
format_op (mile_gpu_dsl/src/mat/gpu_ast.rs (line 910)) 目前只是保留给调试输出的辅助函数，生产流程没有用到，如果想做更完整的调试/日志，可以把它接入。
Compute 阶段仍然限制较多：build_render_expr 不支持同一表达式里引用多个渲染 import，build_compute_expr 依然拒绝 Vec2/3/4 构造器，也未支持循环、自定义函数等复杂语法。
Render-pass 路径里还只有 basic 的 IF / binary / unary 组合，没有封装更高级的 DSL 语法糖（比如直接提供 if_then_else helpers、比较多个 import 等）。
ImportRegistry 里的 import 仍是示例级别，缺少复杂场景（纹理采样、buffer 访问）的 DSL 封装。
如果你接下来要扩展 DSL，可以从补完这些残留点入手，或增强 render/compute 侧的语法能力。
